Como responder com clareza e confiança
01 —
Método PREP
Ponto central (1 frase direta) → Razão (o porquê) → Exemplo concreto do seu trabalho → Ponto central novamente. Evite entrar em detalhes técnicos antes de estabelecer o conceito principal.
02 —
Use analogias do mundo real
"GC é como um zelador que limpa objetos que ninguém mais usa. O stop-the-world é quando ele pede para todos saírem da sala para limpar de vez." Analogias demonstram entendimento profundo, não decoreba.
03 —
Quando não souber a resposta
Diga: "Não trabalhei diretamente com isso, mas pelo que conheço dos conceitos relacionados, acredito que..." Demonstra honestidade e raciocínio. Nunca invente — o entrevistador percebe.
04 —
Conecte ao mundo real
Todo conceito deve ter um "onde isso importou na prática". Ex: "Usamos circuit breaker no serviço de pagamentos para evitar cascade failure quando o gateway externo ficava lento."
05 —
Treine em voz alta
Grave áudios explicando conceitos como se fosse para um colega júnior. Se travar, o conceito ainda não está consolidado. A entrevista é performance — pratique a performance.
06 —
Perguntas para fazer ao entrevistador
"Qual o maior desafio técnico da equipe hoje?"
"Como é o processo de code review?"
"Qual o nível de cobertura de testes do projeto?"

Perguntas boas demonstram maturidade — não são opcionais.
07 —
Estrutura de resposta para system design
1. Confirmar requisitos funcionais e não-funcionais. 2. Estimativa de escala (usuários, requests/s, dados). 3. Componentes de alto nível. 4. Detalhar 1-2 componentes críticos. 5. Trade-offs de cada decisão. Fale em voz alta enquanto pensa.
